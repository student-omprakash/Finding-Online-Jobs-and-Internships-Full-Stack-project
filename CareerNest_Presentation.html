Project Overview — What is CareerNest?

An end-to-end platform connecting students with job opportunities

02 / 11
🎓

For Students

Students can register, build a detailed profile, upload their resume and apply to jobs and internships from hundreds of companies — all from one platform. They get real-time application tracking and email alerts.

🏢

For Recruiters

Recruiters can post job listings with full details, manage and shortlist applicants, update application statuses and view candidate profiles including resume downloads — all in a dedicated dashboard.

🔍

Smart Job Search

Powerful search with filters for keywords, city/location, experience level (Fresher to Expert), and job type (Full-Time, Part-Time, Internship, Remote). Quick search chips on the home page make searching instant.

🔐

Secure Authentication

Industry-standard JWT-based login with role-based access for Students and Recruiters. Forgot password uses a 6-digit OTP sent via Gmail SMTP with a 10-minute expiry — fully secure and verified.

Why DevOps is Needed for CareerNest?

The key reasons DevOps practices make this project better

05 / 11

🐳 Problem — "It works on my machine" but not on others

Without Docker, the app might work perfectly on one developer's computer but crash on another due to different Node.js versions, missing packages or wrong MongoDB setup. Docker solves this by packaging the entire application — code, dependencies, and environment — into containers that run exactly the same everywhere. This removes all environment-related bugs.

🔧 Problem — Manual testing before every deployment is slow and error-prone

Developers would need to manually run lint checks, test the build and verify Docker configs every single time before deploying. Jenkins CI/CD automates all of this — it automatically validates the code quality, runs the build process and checks Docker configuration the moment new code is pushed to GitHub.

📊 Problem — No visibility into how the app is performing in real-time

Without monitoring, we would never know if API responses are becoming slow, if error rates are increasing, or if the server is running out of memory — until users start complaining. Prometheus and Grafana give us live graphs of every API call, response time, CPU usage and error rate, allowing us to detect and fix issues before users even notice them.

🐳

Docker & Docker Compose

How containerization works in this project

06 / 11
🐳

What is Docker doing here?

Docker wraps the Node.js backend and the React frontend into separate containers. The frontend uses a multi-stage Dockerfile — first it builds the React app with Vite, then puts the final files inside an Nginx server container. This means the production image is tiny and fast. The backend container holds all the Express API code and its dependencies.

🔗

What does Docker Compose do?

Docker Compose is like a master controller for all 5 services. It starts MongoDB first, waits for it to be healthy, then starts the Backend, then the Frontend. Prometheus and Grafana also start automatically. All services talk to each other using service names on a shared private network called careernest-network.

📦 All 5 Services Managed

  • 🍃
    MongoDB — Database service on Port 27017 with health check ping
  • ⚙️
    Backend API — Node.js server on Port 5005, starts after MongoDB is healthy
  • 🌐
    Frontend — React app served by Nginx on Port 3000
  • 📡
    Prometheus — Metrics collector on Port 9090, scrapes backend every 5 seconds
  • 📊
    Grafana — Dashboard UI on Port 4000, auto-connected to Prometheus datasource

Prometheus + Grafana Monitoring

Real-time observability and visual dashboards

08 / 11
📡

Prometheus — The Metrics Collector

Prometheus is a time-series database that automatically calls the backend's /metrics endpoint every 5 seconds and stores the data. The backend uses the prom-client library to track every single HTTP request — recording which route was called, the HTTP method used, the response status code, and most importantly, how long it took to respond. This data is stored in histogram buckets ranging from 0.1 seconds all the way to 5 seconds.

Grafana — The Visual Dashboard

Grafana connects to Prometheus as its data source and turns all the raw numbers into beautiful, real-time charts. The careernest_dashboard.json file is automatically loaded when Grafana starts — so the dashboard is ready immediately without any manual setup. You can see live graphs for API request rate, average response time, error rate per endpoint, and system resource usage. This makes it easy to spot problems instantly.
